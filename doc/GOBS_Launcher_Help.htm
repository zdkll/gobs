Launcher 启动界面

1.       概述

Launcher 为GOBS 软件启动界面，提供三个方面的功能：

1)        创建、打开和管理Project 项目的功能；

2)        启动模块功能，共有三个模块：LMGR观测系统模块、RNM节点管理模块、DMC数据管理和QC模块。模块启动需要先选择打开一个项目，启动模块后，模块相关操作的文件都在该项目下，比如工区Area文件夹，数据库，工作目录work等。

3)        菜单Tools 提供扩展工具，目前提供SPS 文件生成工具，即将现场施工文件，包括炮时文件，GOBS投放记录等生成SPS文件。其他扩展开发的工具都可以存放在该菜单下。

2.       使用说明

1)        创建、打开和管理项目Project

   

New Project 按钮和File 菜单New Project 选项

点击后弹出创建项目对话框，如下图，输入项目名称，选择项目文件的目录后，点击 Ok 按钮即可创建一个Project，创建后，默认打开新创建的工区。

Open Project 按钮和File 菜单Open Project 选项

点击后从文件系统中选择项目文件 .gpro 文件打开对应项目。

Recent Projects 列表

记录历史打开的项目，选择某个项目双击或者右键Open，即可打开项目，右键菜单clear 清除项目。

File 菜单Recent Projects 选项

    该选项展示历史项目，可以选择其中某个项目打开，或者点击Clear Menu，清除所有的历史记录。

 

2）启动模块

启动模块入口有两处：1）主界面点击某个模块直接起动；2）Module 菜单下点击模块对应的选项启动模块，如果当前没有打开Project项目，需要先打开项目后再启动模块。

3.       附录

1)      生成SPS文件施工文件格式和命名规则说明

炮时记录文件和GOBS投放记录文件，都是以每条线（炮线 或者 接收线）为一个文件。每个文件对应一条线，文件命名规则如下：

 

炮时文件： ShotLine.SEQXXX.dat 形式， XXX 表示炮线号，如 ShotLine.SEQ001.dat 表示第一条炮线炮时记录文件；

GOBS投放记录文件：RecvLine.SEQXXX.dat 形式， XXX表示接收线号,如 RecvLine.SEQ001.dat 表示第一条接收线投放记录文件。

 

文件格式：

炮时记录文件格式如下:

炮号        y坐标(N)           x坐标(E)         水深     炮时

 

GOBS 接收站投放记录文件格式如下：

Y坐标（N）              X 坐标（E）      GOBS设备号

目前默认接受站号从 1-n，n为该条线的站数。GOBS设备号在LMGR 投放显示列表中，可以读取该文件加载设备投放对应的接收点。

另外，这两种文件格式可以更改和补充，更改后，只需要修改 Export SPS Files 工具即可。
